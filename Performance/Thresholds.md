Es una funcionalidad que permite definir las expecativas de rendimiento del sistema sometido a prueba.
Tresholds establecen el criterio por el cual un test se considera correcto o un error.

Ejemplos de expectativas (Thresholds)

1. El sistema no produce mas de 1% de errores
2. El tiempo de respuesta para el 95% de las solicitudes debe ser inferior a 200 ms
3. El tiempo de respuesta para el 99% de las solicitudes debe ser inferior a 400 ms
4. el punto final especifico debe responder siempre antes de 200 ms
5. Cualquier condición en una métrica personalizada

Thresholds analizan las métricas de rendimiento y determinan el resultado final de la prueba (pasa / no pasa). Thresholds son esenciales para la autmatización de las pruebas de carga
